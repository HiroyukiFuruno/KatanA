## Why

KatanAのUI (`katana-ui`) は現在、テキストエディタ機能（入力、カーソル管理、行番号、構文ハイライト等）を直接抱え込んでいます。v0.26.0でのPreview分離に続き、v0.27.0ではエディタ機能も完全に分離されたクレート（`katana-editor`）へと切り出します。
これにより、KatanA本体のUIドメインから独立した、純粋で高性能なエディタウィジェットが実現され、CI/CDの高速化、単体テストの容易化、そしてマルチプラットフォーム対応の複雑性をそれぞれのドメインに閉じる（Fat UIの解消）ことが可能となります。

## What Changes

- `katana-editor` クレートの切り出し：テキスト入力、カーソル管理、行番号描画、構文ハイライトなど、KatanAのドメイン知識を持たない汎用的な「純粋な高性能エディタウィジェット」の分離。
- `katana-ui` のエディタ責務剥奪：テキスト入力やハイライトの詳細ロジックを排除し、エディタウィジェットを利用する「糊」としての役割へ完全に特化。
- データ駆動アーキテクチャの確立：設定（テーマや絵文字対応フォントの設定など）を `EditorConfig` 等のインターフェース経由で注入し、循環参照を防ぐ。

## Capabilities

### New Capabilities
- `editor-component`: テキスト入力、行管理、構文ハイライトを担う独立したエディタウィジェットアーキテクチャ。

### Modified Capabilities
- `ui-architecture`: Fat UI から「コンポーネントの組み立て」へ特化する UI アーキテクチャの変更（エディタ分離版）。

## Impact

- `crates/katana-ui`: エディタに関する大規模なコード削除とコンポーネント利用への置き換え。
- ワークスペース構成: 新規クレート (`katana-editor`) の追加。
- CI/CD: 独立したインテグレーションテストが可能になり、CIの並列実行効率がさらに向上。
