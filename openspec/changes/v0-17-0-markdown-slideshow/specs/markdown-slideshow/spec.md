## ADDED Requirements

### Requirement: Markdown スライドショーはアクティブ文書を全画面で表示する

システムは、アクティブな Markdown 文書を全画面スライドショーとして表示できなければならない（MUST）。

#### Scenario: Markdown 制御群からスライドショーを起動する

- **WHEN** ユーザーが Markdown 系の制御群に追加されたスライドショー起動ボタンをクリックする
- **THEN** 現在の active Markdown 文書が全画面スライドショーで開かれる

#### Scenario: スライドショーは全画面で表示される

- **WHEN** スライドショーが起動する
- **THEN** 表示領域は全画面になる
- **THEN** 通常の preview pane は背後に隠れる

### Requirement: Markdown スライドショーは左右のページングで移動できる

システムは、スライドショー中に左右のページング操作で前後のページへ移動できなければならない（MUST）。

#### Scenario: 次のページへ移動する

- **WHEN** ユーザーが右方向のページング操作を行う
- **THEN** スライドショーは次のページを表示する

#### Scenario: 前のページへ移動する

- **WHEN** ユーザーが左方向のページング操作を行う
- **THEN** スライドショーは前のページを表示する

#### Scenario: ページングは端で安定している

- **WHEN** ユーザーが先頭ページでさらに前へ移動しようとする
- **THEN** スライドショーは先頭ページのまま維持される
- **WHEN** ユーザーが末尾ページでさらに次へ移動しようとする
- **THEN** スライドショーは末尾ページのまま維持される

### Requirement: Markdown スライドショーは終了導線を明確に持つ

システムは、スライドショーを `Esc` と画面右上の `[x]` から終了できなければならない（MUST）。

#### Scenario: Esc で終了する

- **WHEN** ユーザーがスライドショー表示中に `Esc` を押す
- **THEN** スライドショーは終了する

#### Scenario: 右上の x で終了する

- **WHEN** ユーザーが画面右上の `[x]` をクリックする
- **THEN** スライドショーは終了する

### Requirement: Markdown スライドショーは現在の theme を継承する

システムは、スライドショーの配色と文字色に現在の theme を継承しなければならない（MUST）。

#### Scenario: theme を切り替えた後にスライドショーを開く

- **WHEN** ユーザーが現在の theme を変更したあとでスライドショーを開く
- **THEN** スライドショーの背景色、文字色、強調色はその theme に一致する

### Requirement: Markdown スライドショーは diagram を自動ページ分割する

システムは、diagram を含む Markdown 文書でも印刷時と同様の自動ページ分割でスライドショーを生成しなければならない（MUST）。

#### Scenario: diagram を含むページでも切れ目が自動で決まる

- **WHEN** Markdown 文書に diagram block が含まれている
- **THEN** スライドショーは diagram 専用の手動調整なしでページを分割する
- **THEN** ページの切れ目は印刷時と同様の自動レイアウトに従う
