## MODIFIED Requirements

### Requirement: 検索結果のノイズ削減

システムは、ワークスペースのコンテンツ検索結果を表示する際、コードベース内の定型的なアトリビュート（特に Rust の `#[allow(...)]` など）が検索の主目的でない限り、これらを結果から除外または折り畳んで表示しなければならない（SHALL）。

#### Scenario: allowアトリビュートのフィルタリング

- **WHEN** ユーザーが "my_function" を検索した時
- **THEN** ヒットした行が `#[allow(dead_code)]` などのアトリビュート行であり、かつクエリが "allow" を含んでいない場合
- **THEN** その行は検索結果リストから除外される
