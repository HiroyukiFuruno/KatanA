# 次世代 i18n フレームワーク (fluent-rs/ICU) への移行

## 1. 背景・目的

現在の Katana の国際化（i18n）基盤における `I18nOps::tf` は、単純な文字列置換 (`.replace()`) に基づいています。この設計には以下の致命的な制約があります：

- **複数形（Pluralization）の欠如**: 「1つのファイル」と「2つのファイル」で名詞や動詞が変化する言語（英語、フランス語など）に対応できません。
- **文法的性（Grammatical Gender）の無視**: スペイン語やドイツ語など、対象の性別によって形容詞や冠詞が変化する言語の自然な翻訳が困難です。
- **複雑なメッセージ構築の限界**: 条件分岐や数値フォーマットなどの高度なメッセージ制御が不可能です。

本提案は、Katana を本格的なデスクトップアプリケーションとしてグローバル展開するため、業界標準の `fluent-rs` (Project Fluent) または ICU メッセージフォーマットベースのエンジンへ移行することを目的とします。

## 2. 主な変更点

- **i18n エンジンの刷新**: 単純置換から、言語学的ルールを解釈可能なエンジンの導入。
- **フォーマットの移行**: 既存の JSON ロケールファイルを、より表現力の高い `FTL` (Fluent Translation List) 形式に変換。
- **API の高度化**: `tf` 関数を、単なる文字列リストではなく、名前付き引数（HashMap等）を受け取り、複数形分岐を自動処理する形式にアップグレード。

## 3. 期待される成果

- すべてのサポート言語において、ネイティブスピーカーにとって違和感のない、正確な文法的表現が可能になる。
- 翻訳データの管理が容易になり、将来的な新規言語追加時の品質が向上する。
- 数値、日付、通貨などのロケール固有のフォーマット対応への道筋がつく。
