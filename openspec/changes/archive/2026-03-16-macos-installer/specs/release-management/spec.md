## ADDED Requirements

### Requirement: チェンジログの自動生成

システムは Git のコミット履歴（Conventional Commits などのフォーマット）をスキャンし、`CHANGELOG.md` を自動または半自動で生成・追記しなければならない（MUST）。バージョンタグが一つも存在しないファーストリリースの場合でも、リポジトリの全コミット履歴から初版のチェンジログを正しく生成できなければならない（MUST）。

#### Scenario: チェンジログの更新（既存タグあり）

- **WHEN** 開発者が `make changelog` 等の生成コマンドを実行したとき
- **THEN** `git-cliff`（または代替ツール）が起動し、前回のバージョンタグ以降のコミットログがパースされ、`CHANGELOG.md` の冒頭に追記・フォーマット形式として出力される。

#### Scenario: ファーストリリース時のチェンジログ生成（タグなし）

- **WHEN** リポジトリに過去のバージョンタグが一つも存在しない状態で、開発者が初回のリリースコマンド（`make release VERSION=0.0.1`）を実行したとき
- **THEN** リポジトリの全コミット履歴が対象としてパースされ、初版の `CHANGELOG.md` が正しく生成される。

### Requirement: リリース自動化パイプライン (バージョン管理とタグ付け)

ビルド自動化システム (Makefile) は、ワークスペースのバージョンを上げ、チェンジログを更新し、Git のリリース・コミットおよびタグ付けまでを一括で行うターゲットを提供しなければならない（MUST）。

#### Scenario: バージョンタグ打ちとリリースワークフロー

- **WHEN** 開発者が `make release VERSION=x.y.z` のように新しいバージョン番号を指定してコマンドを実行したとき
- **THEN** 全ての `Cargo.toml`（ワークスペースルート含む）のバージョンが書き換わり、チェンジログが更新され、Git で一括コミットと `vx.y.z` のタグが打たれた上で、ディストリビューションファイル（`.dmg` 等）のビルド処理へとシームレスに連携する。
