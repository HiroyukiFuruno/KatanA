## Why

現在、Katana は `cargo run` コマンドを通じて実行されており、Rust の開発環境が必要です。これでは、一般的な macOS ユーザーが手軽に利用できません。ユーザーに簡単に配布し、ドラッグ＆ドロップでインストールできるようにするため、macOS 標準のディスクイメージ（`.dmg`）を作成する必要があります。

さらに、継続的なリリースとユーザーへの周知を適切に行うための「バージョンタグの管理」と「チェンジログの自動生成」、一般ユーザー向けの「インストールガイド」の提供が必要不可欠です。

加えて、Katana は **オープンソースソフトウェア (OSS)** として公開する予定です。OSS化に伴い、パブリックでの CI/CD（GitHub Actions）のセキュリティ（悪意のあるPRによるリリース処理の自動実行防止など）や権限モデルの整備を前もって行い、安全にOSSコミュニティへ公開する準備を整える必要があります。

## What Changes

- macOS アプリケーションバンドル構造 (`Katana.app`) のパッケージング環境を構築します（`cargo-bundle` またはシェルスクリプトによる手動構築）。
- その `Katana.app` をラップし、ドラッグ＆ドロップインストール用の `.dmg` ファイルを生成します。
- `git-cliff` ツールを導入し、コミット履歴から `CHANGELOG.md` を自動生成する仕組みを構築します。
- バージョン番号のバンプ、チェンジログの生成、Gitタグの作成から DMG 生成までを一括して行うリリースフローを追加します。
- 開発者向け手順を `docs/` 配下に集約し、ルートの `README.md` を「一般ユーザー向けインストール解説」に再構築します。
- **OSS向けCI/CDセキュリティ強化と公開化**: 既存の GitHub Actions CI ワークフロー（`.github/workflows/ci.yml`）と分離する形で CD ワークフローを新設し、オーナー自身による承認/トリガーでのみリリース処理が実行されるよう制限します。そして、パイプライン完成の最終タスクとしてリポジトリを Private から Public（OSS化）に切り替えます。

## Capabilities

### New Capabilities
- `macos-packaging`: macOS 向けの `.app` バンドルおよび配布用の `.dmg` ファイルを生成する機能の追加。
- `release-management`: バージョンタグ管理、チェンジログ自動生成、リリースのフロー自動化機能の追加。
- `oss-transition`: OSS化に向けたCI/CD権限強化、ドキュメントのユーザー向け再構築、パブリック公開タスクの整備。

### Modified Capabilities

## Impact

- `Makefile`: パッケージング（`package-mac`, `dmg`）およびリリース自動化（`release`）用のターゲットが追加されます。
- `crates/katana-ui/Cargo.toml`: バンドル設定（`[package.metadata.bundle]`）の追記。
- `.github/workflows/`: 既存の `ci.yml`（テスト・リント）とは別に、リリース用の CD ワークフローが新設されます。
- `cliff.toml`: チェンジログ生成用の設定ファイルが追加されます。
- `CHANGELOG.md`: ルートディレクトリにチェンジログが追加・運用されるようになります。
- `README.md` / `README.ja.md`: 一般ユーザー向けに全面書き換え。
- `docs/`: 既存の `coding-rules.ja.md` 等に加え、開発環境セットアップガイドが集約されます。
