# Delta Spec: Document Organization — タブグループ描画と操作の改善

## Changes

### Requirement: グループの描画順序（変更）

タブバーの描画順序をグループ優先に変更する。The system SHALL conform.

#### Scenario: グループブロックがピン留めタブより左に描画される

- **WHEN** タブバーにグループとピン留めタブの両方が存在する
- **THEN** グループブロック（ヘッダー + メンバー）が最も左に描画される
- **AND** ピン留めタブ（グループ未所属）がその次に描画される
- **AND** 通常タブ（グループ未所属・ピン未設定）が最も右に描画される

### Requirement: グループヘッダーの視覚的差別化（新規）

グループヘッダーを通常タブと明確に区別できるデザインにする。The system SHALL conform.

#### Scenario: グループヘッダーのチップデザイン

- **WHEN** グループヘッダーが描画される
- **THEN** グループカラーの円形ドット（●）が名前の左に表示される
- **AND** グループ名が通常タブより小さいフォントサイズ（85%）で表示される
- **AND** グループに所属するタブの下部に 2px のカラーアンダーラインが描画される

#### Scenario: 折りたたみ時のグループヘッダー

- **WHEN** グループが折りたたまれている
- **THEN** ドット + 折りたたみアイコン（▸）+ メンバー数のみが表示される

### Requirement: グループ単位の移動（新規）

グループヘッダーをドラッグしてグループ全体の順序を変更できる。The system SHALL conform.

#### Scenario: グループヘッダーのドラッグ移動

- **WHEN** ユーザーがグループヘッダーをドラッグし、別のグループの前後にドロップする
- **THEN** グループ全体（ヘッダー + 所属メンバー）がドロップ位置に移動する

### Requirement: インラインリネーム（新規）

グループヘッダーをダブルクリックしてインラインリネームできる。The system SHALL conform.

#### Scenario: ダブルクリックでリネームモード開始

- **WHEN** ユーザーがグループヘッダーをダブルクリックする
- **THEN** グループヘッダーが TextEdit に置き換わり、現在のグループ名が選択状態で表示される

#### Scenario: Enter キーでリネーム確定

- **WHEN** リネームモードで Enter キーを押す
- **THEN** 入力された名前でグループ名が更新される
- **AND** リネームモードが終了する

#### Scenario: フォーカス離脱でリネーム確定

- **WHEN** リネームモードで TextEdit 以外をクリックする
- **THEN** 入力された名前でグループ名が更新される
- **AND** リネームモードが終了する

#### Scenario: 空名前の場合は元に戻す

- **WHEN** リネームモードで名前を空にして確定する
- **THEN** グループ名は変更されず、元の名前が維持される
