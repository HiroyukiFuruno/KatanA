# Delta Spec: Tab Context Menu — タブグループ UX 改善

## Changes

### Requirement: タブグループ作成フロー（変更）

タブグループ作成時に名前入力と色選択を必須とする。The system SHALL conform.

#### Scenario: グループ作成時に名前入力が必須

- **WHEN** ユーザーがタブのコンテキストメニューから「新しいグループを作成」を選択する
- **THEN** 名前入力欄と色パレット（7色）を含むサブメニューパネルが表示される
- **AND** 名前が空の場合は「作成」ボタンが無効化される
- **AND** 名前を入力し色を選択して「作成」を押すとグループが作成される

#### Scenario: グループ作成時のデフォルト色

- **WHEN** グループ作成サブメニューが表示される
- **THEN** 先頭の色（Blue: #4A90D9）がデフォルトで選択されている

### Requirement: グループコンテキストメニューの拡充（新規）

グループヘッダーの右クリックメニューに追加操作を提供する。The system SHALL conform.

#### Scenario: グループ解散（Ungroup）

- **WHEN** ユーザーがグループヘッダーを右クリックし「グループ解散」を選択する
- **THEN** グループが削除され、所属タブは通常タブとして残る

#### Scenario: グループごと閉じる（Close Group）

- **WHEN** ユーザーがグループヘッダーを右クリックし「グループを閉じる」を選択する
- **THEN** グループに所属する全タブが閉じられ、グループが削除される
- **AND** ピン留めされたタブがグループに含まれる場合は、そのタブはスキップされる

#### Scenario: グループの色変更パレット

- **WHEN** ユーザーがグループヘッダーを右クリックする
- **THEN** コンテキストメニュー内に色パレット（7色の円形ボタン横並び）が表示される
- **AND** 色をクリックするとグループの色が即時に変更される

#### Scenario: グループのリネーム（コンテキストメニューから）

- **WHEN** ユーザーがグループヘッダーを右クリックし「名前を変更」を選択する
- **THEN** グループヘッダーがインラインリネームモードに入る（TextEdit に切り替わる）
