## MODIFIED Requirements

### Requirement: 設定のJSON永続化

全アプリケーション設定は `~/Library/Application Support/KatanA/settings.json` に JSON として永続化され、保存処理は原子的に完了しなければならない（SHALL）。

#### Scenario: 設定の保存

- **WHEN** ユーザーがテーマ、フォントサイズ等の設定を変更する
- **THEN** 変更は一時ファイル経由で `settings.json` に原子的に反映される
- **THEN** 保存失敗時に、部分書き込みされた新ファイルだけが唯一の設定ソースとして残らない

#### Scenario: 設定の復元

- **WHEN** アプリを起動する
- **THEN** 前回正常に保存された設定が自動的に復元される

#### Scenario: 設定ファイルが存在しない場合

- **WHEN** 設定ファイルが存在しない状態でアプリを起動する
- **THEN** デフォルト設定が適用され、最初の保存時に新規ファイルが作成される

#### Scenario: 設定ファイルが破損している場合

- **WHEN** 設定ファイルの JSON が不正な形式である
- **THEN** システムは破損ファイルをバックアップ名へ退避してからデフォルト設定を適用する
- **THEN** ステータスバーに回復可能な警告メッセージが表示される

## ADDED Requirements

### Requirement: 設定保存失敗の可視化

設定保存の失敗は UI から黙殺してはならず、回復可能な失敗として通知されなければならない（MUST）。

#### Scenario: UIからの設定変更で保存に失敗する

- **WHEN** 権限不足やディスクエラーで `settings.json` の保存に失敗する
- **THEN** 現在セッション中の in-memory 設定値は維持される
- **THEN** ユーザーに保存先パスを含む回復可能エラーが表示される
- **THEN** 失敗はログに記録される
