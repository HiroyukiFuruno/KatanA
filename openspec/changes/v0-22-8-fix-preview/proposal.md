## Why

現在、Problems パネルや今後の LLM 自動修正からファイル単位の修正を実行する際、実際にどの差分がファイルへ反映されるのかを確認してから取り込む導線がありません。単一の DiagnosticFix だけでなく、ファイル全体、複数ファイル、ワークスペース全体の修正を安全に扱うには、git diff に近い差分確認画面を必ず挟み、ユーザーが「キャンセル」または「修正を反映」を選べる必要があります。

## What Changes

- ファイル単位の修正適用前に、git diff 風の差分確認画面を表示する。
- 差分確認画面では、右下に「キャンセル」と「修正を反映」を配置し、ユーザーが差分を確認してから取り込めるようにする。
- 差分表示は Split（左右分割）と Inline（行内表示）の2種類を提供し、初期値は Split とする。
- 差分表示方式は設定へ永続保存でき、差分画面上では一時的に切り替えられるようにする。
- 複数ファイルやワークスペース全体の一括修正では、ファイルごとの差分をページ送りのように順番に表示し、ファイル単位で反映または拒否できるようにする。
- Problems パネルの単体「修正」ボタンに対する軽量なホバープレビューは、副次機能として残す。

## Capabilities

### New Capabilities
- `diagnostic-fix-preview`: DiagnosticFix や将来の LLM 自動修正によるファイル差分を、反映前に確認・承認できる機能

## Impact

- `crates/katana-ui`: Problems パネル、修正適用フロー、差分確認画面、設定画面の変更
- `crates/katana-platform`: 差分表示方式の永続設定追加
- `crates/katana-linter`: （必要に応じて）Fix 適用前の差分生成に必要なデータの提供ロジック拡張
