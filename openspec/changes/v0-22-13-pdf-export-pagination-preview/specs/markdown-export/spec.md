## MODIFIED Requirements

### Requirement: PDF出力

現在開いているMarkdownをPDFファイルとしてエクスポートする。システムは、PDF保存前にページ構成を確認する専用プレビューを表示し、ユーザーが1ページ出力または複数ページ出力を選択できなければならない（MUST）。

#### Scenario: PDF出力前プレビューを開く

- **WHEN** ユーザーがExportパネルから「PDFとして出力」を選択する
- **THEN** システムは保存先ダイアログを直接表示しない
- **THEN** システムは `Katana://PDFExportPreview/...` の専用タブを開く
- **THEN** 専用タブはページ一覧、PDFページ見た目のプレビュー、ページ設定、保存操作を表示する

#### Scenario: ページモードを選択する

- **WHEN** PDF出力前プレビューを開く
- **THEN** システムはページモードの既定値を「複数ページ」にする
- **THEN** ユーザーは「1ページ」と「複数ページ」を切り替えられる
- **THEN** システムは選択したページモードに応じてプレビューを更新する

#### Scenario: 1ページPDFとして保存する

- **WHEN** ユーザーがページモードで「1ページ」を選択する
- **AND** 保存操作を実行する
- **THEN** システムは現在のMarkdown全体を1ページのPDFとして保存する

#### Scenario: 複数ページPDFとして保存する

- **WHEN** ユーザーがページモードで「複数ページ」を選択する
- **AND** 保存操作を実行する
- **THEN** システムはページ計算結果に従って複数ページのPDFを保存する
- **THEN** 保存されたPDFのページ構成は出力前プレビューと一致する

#### Scenario: ページ区切り候補を生成する

- **WHEN** システムが複数ページのPDFプレビューを計算する
- **THEN** システムは見出し前、図の前、段落境界からページ区切り候補を生成する
- **THEN** 各候補は有効または無効の状態を持つ
- **THEN** システムは候補線をPDFページ見た目のプレビューと候補一覧に表示する

#### Scenario: ページ区切り候補を選択する

- **WHEN** ユーザーがページ区切り候補を有効または無効にする
- **THEN** システムは同じ設定でページ計算結果を更新する
- **THEN** システムはページ一覧とPDFページ見た目のプレビューを更新する
- **THEN** 無効化された候補ではページを切らない

#### Scenario: 強制分割が必要なブロックを扱う

- **WHEN** 図または段落が1ページの高さを超える
- **THEN** システムはユーザー選択に関係なく、そのブロックを強制分割対象として扱う
- **THEN** システムは強制分割が通常のページ区切り候補とは異なることをプレビュー上で区別できるようにする

#### Scenario: 外部ツール未インストール時

- **WHEN** PDF生成に必要な外部ツールがインストールされていない
- **THEN** エラーメッセージとインストールガイドが表示される
