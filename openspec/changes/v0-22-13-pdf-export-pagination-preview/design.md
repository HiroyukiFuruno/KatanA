## Context

現在のPDF出力は、ExportパネルでPDFを選ぶと保存先選択へ進み、出力結果のページ構成を事前に確認できない。PDF生成側はMarkdownをネイティブ文書表現へ変換し、最終的に画像化した結果をPDFページへ配置しているため、出力前にページ境界を扱うには画像化前のレイアウト情報をUIとPDF生成の共通入力にする必要がある。

この変更では、PDF出力前に `Katana://PDFExportPreview/...` の仮想タブを開き、ページモード、ページ区切り候補、出力前プレビューを同じ画面で扱う。

## Goals / Non-Goals

**Goals:**

- PDF出力前に `1ページ` と `複数ページ` を選択できるようにする。
- 複数ページ時に、見出し前・図の前・段落境界から生成したページ区切り候補をユーザーが有効・無効にできるようにする。
- プレビューと実際のPDF出力が同じページ計算結果を使うようにする。
- UI確認用に、PDF出力前プレビュー画面のスクリーンショットまたは動画を `scripts/screenshot` で生成できるようにする。

**Non-Goals:**

- 任意位置への完全手動ページ区切り追加。
- 用紙サイズ、余白、倍率の詳細設定。
- PDF閲覧ビューア機能そのものの拡張。
- PNG/JPEG出力のページ分割対応。

## Decisions

1. PDF出力設定は `PdfExportOptions` として扱い、ページモードは `PdfPageMode::Paginated` を既定値にする。
   - `SinglePage` は文書全体を1ページに収める既存互換の出力として扱う。
   - `Paginated` はページ計算結果に従って複数ページへ出力する。

2. Markdown exportの中間表現は、描画ブロックごとの高さとページ区切り候補を持つ。
   - 候補種別は `Heading`, `Diagram`, `Paragraph` を最小セットにする。
   - 候補は安定したID、表示ラベル、文書内の概算位置、有効状態を持つ。
   - ページ高さを超えるブロックは候補設定に関係なく強制分割対象にする。

3. ページ計算は `katana-core` 側の単一サービスに集約する。
   - UIプレビューはこの計算結果を読むだけにする。
   - PDF出力も同じ計算結果を使い、UIと保存結果のズレを避ける。

4. PDF出力前プレビューは仮想タブで実装する。
   - ExportパネルのPDFボタンは保存ダイアログを直接開かず、アクティブ文書から `Katana://PDFExportPreview/<document-id>` を開く。
   - 仮想タブ内にページ一覧、中央プレビュー、ページモード切り替え、候補線リスト、保存ボタンを配置する。
   - 保存ボタンだけがファイル保存ダイアログを開く。

5. ページ区切り候補は「候補線を選ぶ」方式に限定する。
   - ユーザーは候補線の有効・無効だけを変更する。
   - 任意位置追加やドラッグ移動は仕様外にして、v0.22.13の実装範囲を安定させる。

## Risks / Trade-offs

- [Risk] 画像化前レイアウトと最終PDFの見た目がずれる可能性がある → ページ計算結果をプレビューとPDF生成で共有し、ズレを回帰テストにする。
- [Risk] 候補線が多すぎると操作が重くなる → 既定表示は見出しと図を優先し、段落境界はリスト上で簡潔に表示する。
- [Risk] 長い図や長い段落はユーザー選択だけでは収まらない → ページ高さを超えるブロックは強制分割として明示し、候補線とは別扱いにする。
- [Risk] Exportの既存操作から1クリック増える → PDFだけ保存前プレビューを挟み、HTML/PNG/JPEGの操作は変更しない。
