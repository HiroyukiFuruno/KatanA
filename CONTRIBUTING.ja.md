# KatanA Desktop への貢献（コントリビューション）

KatanA Desktop への貢献をご検討いただき、ありがとうございます！
あなたのような方々の力で、このツールはより良いものになっていきます。

> **Note**: このファイル（`CONTRIBUTING.ja.md`）は英語の正本版である `CONTRIBUTING.md` の日本語対訳版です。更新を行う際は、必ず両方のドキュメントを同期（同時更新）するようにしてください。

## 私たちが求めていること

KatanA はまだ初期段階にあり、その未来を現在進行形で形作っているところです。特に以下のような領域での協力を歓迎します：

### 1. デザインと UX へのフィードバック

KatanA を「速い」だけでなく「美しく直感的」なものにしたいと考えています。

- デザインモックアップ、ワイヤーフレーム、CSS やスタイリングの提案。
- ユーザー体験 (UX) をより開発者フレンドリーにするためのフィードバック。

### 2. プロダクトの方向性と「壁打ち」

「何を追加し、何を捨てるか」「何を優先すべきか」を一緒に考えてくれる方を求めています。

- 現代の Markdown ワークスペースに不可欠な機能とは何か？
- 削除したり、もっとシンプルにすべき既存機能はどれか？
- ディスカッションに参加し、ロードマップの優先順位付けにぜひご協力ください。

### 3. AI エージェント支援開発への参加

このプロジェクトは **AI エージェント・ファースト** の手法（主に Antigravity を使用）で構築されています。

- AI コーディングエージェントを活用して開発を行うコントリビューターを強く歓迎します！
- AI エージェントによって生成・支援された Pull Request も大歓迎です。（ただし、実装の最終検証は人間が行ってください。）

## 参加方法

### アイデア、質問、フィードバックがある場合

**[GitHub Discussions](https://github.com/HiroyukiFuruno/KatanA/discussions)** にご参加ください。ここは、プロダクトの方向性を話し合ったり、新機能を提案したり、アイデアの壁打ちを行うのに最適な場所です。

### バグを発見した場合

**[GitHub Issue](https://github.com/HiroyukiFuruno/KatanA/issues)** を起票してください。その際、可能な限り詳細な情報（macOS のバージョン、KatanA のバージョン、再現手順など）をご記入ください。

### コードを書きたい場合

1. まずは既存の Issue や Discussion を確認してください。
2. もし AI エージェント（Antigravity など）を利用する場合は、リポジトリ内の `.agents/skills/` ディレクトリを読み込ませることで、当プロジェクトのワークフローをエージェントに事前学習させてください。
3. ローカルのセットアップ手順やコーディング規約については、**[Development Guide](docs/development-guide.md)** および **[Coding Rules](docs/coding-rules.md)** をそれぞれお読みください。
4. Pull Request を送信してください。

## 同期ルール

このガイドラインを更新する際は、必ず `CONTRIBUTING.md`（英語版）と `CONTRIBUTING.ja.md`（日本語版）が内容的に合致するように同時修正を行ってください。

KatanA をより良い刀へと鍛え上げるためのご協力に感謝します！
